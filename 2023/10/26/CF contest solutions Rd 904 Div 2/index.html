Problem A: Simple Design

find the minimum integer bigger than x and the sum of all its digits is divisible by k

Noting that the sum of all digits is always changing, and will roughly cover every value equally. So a simple brute force search will be fast enough for a nine-digit answer!

That took me 5 mins XD

Problem B: Haunted House - A pretty interesting question.

One key finding: if ${2^i}$ | x , then denote X as x’s binary expression. X won’t have 1s in the (i-1) lowest positions.

Second key finding: if there are more 0 slots in the legit slots, and fewer 1s in the illegal slots (the division line is the ith position), then we can definitely swap to make the number divisible by ${2^i}$

Third key finding: the best swapping strategy is equivalent to move every illegal ‘1’ to the closest legal ‘0’ slot.

Implementation: Noting that the time complexity won’t suffice to calculate it independently for every i, we need to update the answer as we go over every value.

for i from 0->n

#Case1: if we encounter a ‘1’, it means there’s one more 1 to be swapped

#Case2: if we encounter a ‘0’, it means that the zero can not be legal slot anymore, we need to find a new 0 to fill the ‘1’ initially filled by this ‘0’

This one took me a lot of time bruh

Problem C: Medium Design

You initially have a series with length n. Each value is 0. Given a set of segments [l, r], choose an arbitrary number of the segments. For each chosen segment, add 1 to every value covered in the segment. Find the maximum value of the (biggest value - smallest value)

One key finding: Suppose that the biggest value is the $x^{th}$ value in the series. Then for each segment containing this value, choosing it won’t hurt the best outcome, not choosing it might.

Second key finding: Given the first finding, it is intuitive to realize that the first / last value will be the minimum in the end.

Implementation:

  1. go through all the segments that do not include the first value. Calculate the answer
  2. go through all the segments that do not include the last value. Calculate the answer.
  3. Find the max of the ans
