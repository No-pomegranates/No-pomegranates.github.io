F. New Year’s Puzzle – a pretty interesting yet concise algorithm

The first thought is definitely DP… But It’s quite hard to actually think it through, as dp relies on one core element: no aftereffect. However, putting a tile horizontally will influence the next column.

So now we switch to another perspective: greedy algorithm?

rationale? greedy and dp are usually always linked with each other XD

we go through some potential scenarios: ↓

we have some observations:

  • if two vertical columns are both empty, it is always good to put a vertical tile,
  • if one of them is blocked, we have only one option which is to put a horizontal tile
  • there are fixed rules when putting horizontal tiles, because once you put one, you have to continually form this ‘z’ shape

so now we have a general idea:

  1. fill all vertical columns from the two sides until we encounter a filled tile
  2. form z shapes until we are faced with another vertical tile option
  3. break whenever it’s impossible

so how do we determine impossibility?

A nice find: in order to form z shapes, the adjacent filled tiles should be opposite color when painting the board like a chess board, in other words, A(x,y) B(c,d) x+y and c+d should be opposite in parity

G. Moving to the Capital – the only challenging problem

this is a really interesting problem, as it tests your ability to transfer learnt algorithm and fit them in new occasions.

  • we first run a BFS to calculate minimum distance from the capital. (easy piece of cake)

  • then we form a graph consisting only of edges a -> b when d[a] < d[b] (the edges that can be travelled infinitely many times)

  • we are ONLY ALLOWED TO MOVE BACKWARDS ONCE, does this sound familiar?

  • TARJAN algorithm https://www.youtube.com/watch?v=wUgWX0nc4NY
